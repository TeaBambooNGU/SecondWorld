# Product Spec - 多 Agent 小说草稿

## 目标
- 通过多 Agent 协作生成可读性强、AI 味低的玄幻章节草稿。
- 由导演 Agent 控制节奏、冲突与统一文风。
- 每章输出 Markdown，可断点续写。

## 用户
- 主要用户：单人作者（VSCode 写作）。

## 范围
- MVP：本地 CLI 流水线、DeepSeek API、多 Agent 编排。
- 不包含：Web UI、发布平台对接、自动排版成书。

## 功能
- 角色配置（约 20 个角色 + 若干性格模板）。
- 章节流水线：大纲 → 计划 → 角色贡献 → 导演合成。
- 文风指南与低 AI 规则内置到提示词。
- 章节 Markdown 输出与状态跟踪。

## 用户流程
- 提供大纲 → 选择章节 → 生成计划 → 多 Agent 对话 → 导演合成 → 用户修订。

## 数据与集成
- YAML 配置：project/agents/outline。
- Markdown 文风指南。
- DeepSeek API。

## 非功能需求
- 可读性优先，节奏自然，情绪克制。
- 支持断点续跑与日志可追溯。

## 验收标准
- 单章 2000–4000 字。
- 导演能校验冲突与节奏。
- 章节输出为 Markdown，按章归档。
