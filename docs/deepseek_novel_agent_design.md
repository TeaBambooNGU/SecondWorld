# DeepSeek 玄幻小说 Agent 设计文档

## 1. 目标与范围
- 目标：在无人参与微调的前提下，生成 3000-5000 字玄幻小说单章，最大化“去 AI 味”。
- 模式：单次输入，多阶段内部处理；输出包含风格卡、节拍、中间草稿、质检摘要、最终正文。
- 技术栈：LangChain Python + deepseek-reasoner API。
- 日志：纯文本输出（便于记录与回放），不输出代码。

## 2. 约束与原则
- 单次输入包含：风格规则 + 示例段落 + 剧情梗概 + 禁用词 + 字数范围。
- 质量优先，允许自动重写 1-2 次。
- 输出必须包含节拍和中间草稿，便于审计。
- 提示词与解析规则需强约束、低歧义。
- 中文为主，技术术语可保留英文。

## 3. 输入契约（一次性提交）
### 3.1 字段定义
- 风格规则
  - 叙事视角/叙述距离
  - 句式偏好（短句/长句比例、排比/倒装等）
  - 语气与情绪底色
  - 常用词/禁用词
  - 允许/禁止修辞
- 示例段落
  - 2-3 段，合计 300-800 字
- 剧情梗概
  - 时代/背景
  - 角色关系
  - 必须出现的情节点（3-8 条）
  - 结尾方向
- 输出要求
  - 字数范围（3000-5000 字）
  - 章节结构（段落数/分节）

### 3.2 输入模板（建议）
```
[风格规则]
- 视角/叙述距离：
- 句式偏好：
- 语气/情绪底色：
- 常用词/禁用词：
- 允许/禁止修辞：

[示例段落]
- 示例1：
- 示例2：
- 示例3（可选）：

[剧情梗概]
- 时代/背景：
- 角色关系：
- 必须出现的情节点：
- 结尾方向：

[输出要求]
- 字数：3000-5000 字
- 结构：
```

## 4. 输出契约（纯文本日志）
输出按以下顺序组织：
1) 风格卡
2) 节拍表
3) 中间草稿
4) 质检摘要
5) 最终正文

### 4.1 推荐日志格式
```
### 风格卡
...文本...

### 节拍表
1. ...
2. ...

### 中间草稿
...文本...

### 质检摘要
- 禁用词：通过/不通过
- 句首重复率：x%
- 高频套话次数：x
- 节拍覆盖率：通过/不通过
- 结论：通过/重写

### 最终正文
...文本...
```

## 5. 多阶段流程设计
### 5.1 阶段 1：风格提炼
- 输入：风格规则 + 示例段落
- 产出：风格卡（可执行约束）
- 关键点：
  - 句式比例（短/中/长）
  - 叙述距离（贴近/中距/拉远）
  - 词汇密度与用词倾向
  - 禁用表达与替代策略

### 5.2 阶段 2：节拍拆分
- 输入：剧情梗概 + 风格卡
- 产出：8-15 个节拍（动作/转折/情绪推进）
- 关键点：
  - 每个节拍应包含“动作 + 场景 + 情绪推进”

### 5.3 阶段 3：初稿生成
- 输入：风格卡 + 节拍表
- 产出：3000-5000 字初稿
- 关键点：
  - 严格覆盖每个节拍
  - 避免模板化过渡与过度总结

### 5.4 阶段 4：去 AI 味润色
- 输入：初稿 + 风格卡
- 产出：润色稿
- 关键点：
  - 打散句首重复
  - 提升具象细节密度
  - 替换高频套话/连接词

### 5.5 阶段 5：质检评估
- 输入：润色稿 + 风格卡 + 禁用词
- 产出：质检摘要（通过/重写）
- 关键点：
  - 统计与阈值验证
  - 节拍覆盖核验（逐条对照）

### 5.6 阶段 6：自动重写
- 触发：质检不通过
- 次数：1-2 次
- 策略：明确指出失败项，仅重写相关段落或整体重写

### 5.7 阶段 7：输出封装
- 汇总风格卡、节拍表、初稿、质检摘要、最终正文
- 按“输出契约”格式输出

## 6. 关键数据结构（文本字段规范）
### 6.1 风格卡
- 叙述视角：第一/第三（有限/全知）
- 叙述距离：近/中/远
- 句式比例：短句/中句/长句
- 节奏：紧/中/缓（含建议段落长度）
- 常用词：列举 5-10 个
- 禁用词：列举 10-20 个
- 修辞偏好：允许/禁止
- 语气：冷静/克制/热烈等

### 6.2 节拍表
- 编号 + 场景 + 动作 + 情绪推进
- 例：
  - 1. 山门初现，主角试探结界，紧张升级

### 6.3 质检摘要
- 禁用词：通过/不通过（统计次数）
- 句首重复率：x%
- 高频套话次数：x
- 节拍覆盖率：通过/不通过
- 结论：通过/重写
- 备注：需要调整的具体项

## 7. 提示词规范（无代码，仅模板）
### 7.1 风格卡提炼提示词
- 目标：从示例段落中抽取可执行风格约束
- 要求：输出为“风格卡字段清单”，避免空泛描述

### 7.2 节拍拆分提示词
- 目标：将剧情梗概拆为 8-15 个节拍
- 要求：每个节拍必须包含“动作 + 场景 + 情绪推进”

### 7.3 初稿生成提示词
- 目标：严格覆盖节拍表，控制字数
- 要求：避免总结式写法；避免模板化转折词

### 7.4 去 AI 味润色提示词
- 目标：降低模板化表达与重复句式
- 要求：
  - 打散句首重复
  - 替换“似乎/仿佛/不由得/然而/渐渐/于是”等套话
  - 增加具象动作与感官细节

### 7.5 质检提示词
- 目标：输出可判定的指标与通过结论
- 要求：必须给出明确“通过/重写”的结论

## 8. 去 AI 味策略与指标
### 8.1 建议指标
- 句首重复率（前 2 字相同）≤ 8%
- 禁用词出现次数 = 0
- 高频套话（如：似乎/仿佛/不由得/然而/渐渐/于是/忽然）总次数 ≤ 6
- 节拍覆盖率 = 100%
- 段落句长差异明显（长短句混合）

### 8.2 去 AI 味手段
- 增加具体动作链与环境交互
- 避免“情绪解释式”句子堆叠
- 控制抽象形容词密度
- 适度加入符合世界观的物件与细节

## 9. LangChain 组合方式（说明性）
- 结构：
  - 风格提炼链 → 节拍链 → 初稿链 → 润色链 → 质检链
  - 质检不通过时进入重写分支
- 解析：
  - 使用严格字段解析器（或自检修复）
- 重试：
  - 质检不通过可重写 1-2 次

## 10. deepseek-reasoner 调用建议
- 低温任务（风格提炼/质检）：temperature 0.2-0.4
- 生成任务（初稿/润色）：temperature 0.7-0.9
- 上下文限制：优先保证示例段落与风格卡完整输入
- 字数控制：生成任务明确“3000-5000 字”并要求不足时补写

## 11. 异常与失败处理
- 解析失败：要求模型自我修复输出格式
- 字数不达标：补写指定节拍段落
- 风格偏离：回退到“风格卡强化”并重写
- 禁用词命中：只重写包含禁用词的段落

## 12. 扩展点
- 多风格混合：支持多示例权重分配
- 长篇支持：章节级缓存与全局人物语气一致性校验
- 质量评分：引入独立评估模型
